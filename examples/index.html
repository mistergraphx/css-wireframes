Documentation

CSS selectors naming convention

All the framework classes are prefixed with “wf-” : wf-block, wf-note, etc.
Furthermore, we use the BEM naming convention. It stands for Block Element Modifier, and in a nutshell, is a practicall way to make names of CSS selectors as informative and clear as possible. It looks like this : block__element--modifier.
Use modifiers in conjonction to the basic class, like so : class="wf-something wf-something--modifier".
Please note that there's a bit a flexbox, transformations and such, so old outdated browsers are a no no!

Color scheme

The framework is monochrome by choice : the sole purpose of a wireframe is to show the layout of the page, it should not convey any information about its theme. There are a few colored elements available, but they should only be used in very specific cases.

Block

A “block” allow you to materialize sections of your page.
Basically, it's just a border with a bit of padding. 

<div class="wf-block"></div>

Blocks can have a label, which is centered horizontally and vertically. When labelled, a block is supposed to contain only the label, and no other content. 

<div class="wf-block">
	<label class="wf-block__label">A label</label>
</div>

Add diagonals in the background. 

<div class="wf-block wf-block--crossed"></div>

Make the block fill its container. 

<div style="height: 6rem">
	<div class="wf-block wf-block--fill"></div>
</div>

Colored background to highlight some blocks : dark, light, red, green, yellow, pink. 

<div class="wf-block wf-block--color-dark">
	<label class="wf-block__label">Dark</label>
</div>

	<label class="wf-block__label">Light</label>
</div>

</div>

Media

This class turn any <img> into a placeholder media : image, audio file or video.
Ideally the image should be transparent.

For convenience, there are several images provided in the images/ folder, with the most common ratios : square, portrait, landscape, etc.
In short, choose the ratio you like, put a width or a height and you're done.

Note

Use notes to add comments about any elements (typically blocks). They are dispayed on the top right corner of their parent, and can be hidden.

A usefull comment 
<div class="wf-note">A usefull comment</div>

Helpers

Make any element optional so it can be hidden. You can display them again later with a toolbar displayed on the bottom right.

I'm optional, hover me and hide me! 
<div class="wf-block wf-optional">
	<span class="wf-block__label">I'm optional, hover me and hide me</span>
</div>
